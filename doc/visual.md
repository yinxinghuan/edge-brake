# 《急刹车》视觉圣经

## 1. 视觉命题

- 游戏与受众：面向移动端碎片时间玩家的单手反应游戏，首局无需阅读说明。
- 情绪承诺：按住时感受力量不断绷紧，松手后看角色把这一次判断完整兑现，并在崖口前等待距离揭晓。
- 一句话方向：一支可收藏的低多边形极地远征队在断裂冰舌上轮流蓄力冲刺，每个体重、速度、动作、天气和镜头变化都服务于“这次会滑多远”。
- 标志性瞬间：力度槽进入金色高区，角色压低到极限；玩家松手的同一帧角色蹬出雪屑，镜头猛然拉开整条浮岛跑道。
- 必须具备：角色轮廓和体型差异清楚、冰崖有可信厚度与地貌层次、距离易判断、收藏入口不打断核心操作。
- 避免方向：写实摄影、廉价通用卡通 UI、霓虹玻璃科技感。

## 2. 构图与镜头

- 方向与比例：竖屏固定游戏场 390×700，3D 场景填满画布，DOM HUD 内部适配；同时验证 320×568 与 390×844。
- 首屏镜头：使用以角色为中心、位于前侧约 35° 的 `(characterX+6.8, 6.8, 4.8)`、zoom 64 三分之四英雄机位，观察点锁定当前角色胸口；用 3.2 秒完成幅度不超过 0.35u 的轻微侧向漂移。画面需同时看见脸、胸口、左右肢体和朝向跑道的脚尖，角色高度约占游戏场 14–18%，不以背面轮廓或远景展示牺牲角色辨识度。
- 开始换位：封面、下一关和重试都不限时停在角色英雄近景。按下后镜头在最多 1600 ms 内从 zoom 64 缓慢压到 zoom 70，并保持脸、三行角色信息、双臂和脚部完整可见；松手同帧角色移动，相机先跟随角色 280 ms 展示蹬地近景，再用 920 ms ease-out 拉远到以整条跑道中点为观察中心的 `(trackCenter-4.0, 14.0, 21.0)`、zoom 6.8 全景。画面必须同时看见起点、连续完整跑道、崖口和少量崖外海面。到位后固定停留 900 ms，再进入推进。
- 游戏镜头：全景停留结束后将 OrthographicCamera 按危险进度连续推近到角色前侧约 32° 的 `(characterX+5.8, 5.2, 3.6)`、zoom 6.8→68。三种结果保持独立镜头语法：成功定格 120 ms 后绕到悬崖外侧正面 zoom 78；动力不足定格 110 ms 后升高拉远至 zoom 7.4，完整展示剩余跑道；坠崖冻结 140 ms 后沿崖壁垂直追随碎片，zoom 84→62。
- 主要焦点：画面中部 y=280–470 的当前角色、冰面与悬崖边缘。
- 层次：前景风雪、雪屑与近岸碎冰；中景角色、裂缝冰舌与立体断崖；背景为浮冰、冰柱、低对比冰山、天气层，以及只在全景建立阶段浮现的随机巨物剪影。巨物完整模型统一放大到 28–34u 高，并位于跑道后方的深海雾带；模型约 60% 沉在海面与雾层下，只露出足以辨认类别的巨大头部、肩背、角或机械上盖，不得形成比跑道更亮的第二视觉焦点。
- 浮岛厚度：借用悬浮山体“断裂悬挑、根系垂落”的空间节奏，但几何语言严格保持《跳一跳》式方块。跑道的连续薄冰基座下交替出现两条延伸出画框的深长方体、两段画面内结束的中短方体和 6 个短方块；各块宽度、长度、前后深度和间隙不同，形成错落负空间，不做上宽下收。最后 5% 崖口必须保持薄而危险的悬挑。
- HUD 安全区：顶部 28–112 px；底部 566–670 px 为教学/按压提示区，不遮挡赛道。
- 注意路径：角色前进方向 → 明亮冰崖切口与垂直冰壁 → 崖外海面 → 点击反馈与距离结果。

## 3. 色彩

- 天空背景：极夜蓝 `#10263B`；远景冰山 `#1F435A`；雪面 `#EAF9F7`；冰侧 `#A9DCD8`。
- 主文字：暖雪白 `#F7FBF6`；次文字 `#A9C2C5`；墨色轮廓 `#102129`。
- 角色：完整沿用共享素材库 52 名角色的暖中性色、压灰服装色与统一青绿强调 `#3FB6AC`；程序化企鹅退出正式角色池。
- 操作强调：蓄力珊瑚红 `#F06449`，高力度区转为奖励黄 `#FFD166`；安全青绿 `#69D1B5`；失败红 `#E9504F`。
- 使用比例：天空与远景 55%，雪与冰面 25%，角色及轮廓 14%，强调色不超过 6%。
- 禁止：蓝底上用低对比蓝字；同时大面积使用珊瑚红和奖励黄；用颜色作为唯一评价信号。

## 4. 字体

- 展示与数字：`Arial Rounded MT Bold`, `Nunito`, `PingFang SC`, sans-serif；数字使用 tabular-nums。
- 正文与按钮：`Avenir Next`, `PingFang SC`, `Microsoft YaHei`, sans-serif。
- 标题 38/44 px、900；分数 30/34 px、800；按钮 18/22 px、800；辅助标签 13/18 px、700。
- 只允许分数与短标签全大写英文；中文不加宽字距，不使用描边文字。

## 5. 形状、材质与光线

- 主导形状：角色沿用素材库 box / low-segment cyl / icosa / cone / wedge 的体素语言；冰崖由错层盒体、楔形雪脊、六段冰柱和碎冰组成；UI 为 12–18 px 圆角的实心远征标签。
- 边缘：全部 3D 素材使用 flatShading 的硬切面，不加卡通描边；轮廓由三值光照与接触阴影建立。
- 材质：哑光低多边形材质，`metalness=0`、高粗糙度，不使用照片纹理或平滑高模表面。
- 光线：顶部主光、冷色填充光与暖色轮廓光形成 1.0/0.8/0.62 三值面；冰崖切面比顶面深 20–25%，角色和冰面必须有接触阴影。

## 6. 角色、环境与素材

- 角色阵容：完整接入素材库 52 名可扮演角色。模型保持原始轮廓和三值光照，通过统一落脚点进入游戏；角色体积、类别与体态映射为 18–240 kg 的体重，腿长、动作类别与体型映射为 2.6–5.0 的速度。属性必须与肉眼观感基本一致，重甲、熊和牛明显更重，纤细人物和小动物明显更轻。
- 朝向：全部人形及特殊人形的脸部位于本地 +Z，使用 Y=+90° 对齐跑道；全部动物头部位于本地 +X，使用 Y=0° 对齐跑道。首屏为了看脸只能移动相机，禁止把角色本体斜转向镜头。
- 姿态：待机以 0.04–0.10u 起伏和 2–5% 纵向拉伸表现呼吸。蓄力时角色按力度持续下蹲、后摆手臂并压低重心，不能播放一次性下蹲后复位；力度超过 0.82 后加入不超过 0.04u 的紧张颤动。松手后前 620 ms 为助跑/蹬地，人物大幅交替摆臂迈步，四足动物使用冲刺对角足组，鸟类短步扇翼，青蛙同步后腿蹬伸，幽灵前冲拉伸，机甲重踏；随后步幅渐小并进入惯性平衡。失败统一冻结 140 ms 后替换为 14 块低多边形碎片慢速散落。
- 环境：主冰舌至少包含 3 层厚度、2 条可见裂缝、3 组雪脊、5 根冰柱、4 块近岸碎冰和带浪脊的深海；顶面、侧壁、深层冰与无边界海域底色形成四级明度。场景不绘制水平海面网格，避免正交相机近端裁切产生被误读为跑道阴影的直线；海域由全画布底色、崖下浪脊、漂浮碎冰和天气雾共同表达。加长跑道按 0–30%、30–68%、68–100% 分成稳滑、加速、临崖三段，用低对比青蓝冰带和越来越密的短划纹提示速度变化，不使用箭头或文字路牌。远海巨物直接复用 11 个本地素材库 GLB，覆盖大型动物、怪兽、机甲与神话角色；保持原始 low-poly 轮廓，统一施加雾色剪影材质、28–34u 高度与约 60% 的海面下沉，并使用固定世界高度裁切平面彻底移除下部身体；截面依靠低透明度自然软化，不叠加实体雾片。
- 天气：晴朗保持高对比；飘雪使用 120 个慢速点状雪片；薄雾缩短远景雾距但保留角色与崖口；风雪使用 180 个快速斜移雪点和更冷的环境光。约 650 ms 淡入，天气层跟随镜头。
- 游戏资产使用共享 low-poly 库的模型与同源图元，几何吸附 0.125u 网格，统一 flat-shaded 三值语言；不混用照片、平滑 3D 高模、2D 贴片或 emoji。

## 7. UI 与图标

- 图标：自绘 24 px SVG 线性图标，2.2 px 圆头描边；声音、金币、角色和关闭使用同一套轮廓规则并提供可访问名称。
- 主按钮：最小 168×52 px，珊瑚红实心、墨色 2 px 下边阴影；按下下移 3 px并缩放至 0.98。
- HUD：无大面板；顶部显示关卡、分数和金币。滑行阶段恢复紧凑的距离示意，显示“距断崖 N px”、跑道进度条、天气和阻力系数；进入自动刹车或跑道进度超过 72% 后转为珊瑚红警示。标题封面不显示角色属性；进入准备与蓄力阶段后，以无框、无底色、无阴影的三行悬浮文字依次显示姓名、“体重 82kg”和“速度 3.6”，放在角色上方的纯色天空留白，并与角色轮廓保持至少 34 px 间距；角色出发后 180 ms 内淡出。
- 蓄力控件：底部 180×70 px 深色底座，主文案“按住蓄力 / 松手出发”，下方为 0–100% 力度槽；填充由青绿过渡到珊瑚红，高于 82% 出现金色端帽和文字“强”。颜色之外同时使用百分比、长度和角色姿态表达力度。
- 远征队：覆盖层使用暖白考察日志底板，52 名角色以两列滚动卡片排列；卡片包含缩略图、名称、类别、体重、速度、价格和选择状态，不再显示抓地系数。可滚动卡片使用 `onClick`，每个目标至少 92×156 px。
- 状态：焦点使用 3 px 暖雪白外框；禁用降至 42% 不透明度；警告同时用文字、姿态和红色；成功同时显示距离数字、评价文字和粒子。
- Emoji 规则：禁止用 emoji 作为功能图标或游戏角色素材。

## 8. 动效与特效

- 性格：短促、略带橡皮筋弹性；普通 UI 160–220 ms，关卡切换 420 ms。
- 首屏：角色近景持续轻微呼吸与左右重心转移；引导手指必须复用 Google Material Icons `touch_app` 标准路径，不允许重新手绘。图标以 1.5 秒周期完成“抬起—按下—指尖波纹扩散”，首个真实输入后立即消失；减少动态时手指保持静态并显示短标签。
- 蓄力：按下同帧力度槽开始增长、角色压低、镜头缓慢压近；力度越高，动作幅度、槽内脉冲频率与音高越高。达到满值后保持，不闪烁催促。
- 发车与行进：松手同帧角色蹬地、雪屑爆发、三行角色信息和力度槽淡出，先保持 280 ms 跟随近景，再启动 920 ms 拉远；全景保持 900 ms 后再按危险进度推进。前 620 ms 使用助跑动作，之后自然过渡为惯性滑行与平衡动作。滑行过程中没有第二次操作。
- 自动刹车：预计距自然停止剩余 1.25 秒时进入；若角色注定冲出跑道，则最晚在距断崖 0.9 秒时进入，镜头继续按危险进度推进而不跳切。人物恢复旧版后仰、双臂向后上方摆动、双腿错位外撑的刹车动作；鸟类张翼、四足动物压低前爪、机甲后移重心。脚后 9 枚雪屑以 4.5 Hz 循环喷出，姿态与雪屑保持到停车或坠崖；减少动态时保留静态刹车姿势，只取消颤动和循环粒子。
- 随机巨物彩蛋：角色滑出后 80–600 ms 随全景拉远从雾中淡入；动物使用 5.8 秒呼吸起伏，怪兽使用 6.4 秒侧摆，机甲与牛头怪使用更重的 7.2 秒俯仰。巨物必须完整覆盖发车后 1200–2100 ms 的最远全景与停留阶段，2300 ms 后开始淡出、3400 ms 完全消失；连续两局不得出现同一角色。减少动态时取消起伏、侧摆和俯仰，只保留渐显渐隐。
- 输入：按下同帧开始蓄力；松手同帧锁定当前力度并发车。按压、松手、发车三个时刻必须在画面、动作和声音上可区分。
- 收藏：购买成功后金币数字下跳、卡片印章由价格切换为“已拥有”，角色模型在 220 ms 内弹入当前选择；余额不足使用水平 4 px 轻摇并显示差额，不弹通用错误框。
- 停车：180 ms 定住，100 分制成绩在 140 ms 内向上弹入；0–320 px 的过关结算保持不限时，主按钮预告“下一位角色”。超过 320 px 时盖上“挑战无效”红色票据章，说明“你停在了安全观景区 / 未进入断崖警戒线”，以测距虚线贯穿人物到崖口，1450 ms 后才显示“本关重试”。贴边额外有冰崖暖光和 8 枚旋转的低多边形冰晶。
- 失败：140 ms 冻结建立冲击点，180 ms 内角色碎成 14 块，随后约 930 ms 慢速散落；镜头仅在碎裂瞬间轻震一次，位移不超过 4 px、时长 100 ms。
- 粒子：雪屑为 3–7 px 暖白椭圆；奖励冰晶为 4–8 px 黄白菱形，生命周期 220–480 ms。
- 减少动态：移除屏幕震动、背景风带和大量粒子；保留角色姿态、距离数字与评价文本。

## 9. 参考转译原则

- 《跳一跳》与《Block Hop》：采用待机小跳、四肢节奏、天气平滑轮换、金币收藏和即时重开；不复制道路、车辆、方块或原场景构图。
- 定点停车类街机：采用越接近危险边缘奖励越高的风险结构；不使用真实车辆仪表盘。
- 纸雕动画：采用色块分层和轻微材质划痕；不模拟复杂纸张噪声或厚重阴影。

## 10. 反模式

- 禁止 emoji、混合风格图标、写实冰川照片和平滑高模企鹅素材。
- 禁止显示旧版 Aigram / Alter 角落水印；游戏画面不叠加任何平台废弃品牌标记。
- 禁止玻璃拟态、霓虹光晕、大面积渐变、通用浮动白卡和无意义阴影。
- 禁止回到“单块白板”冰面；禁止让雪屑遮住冰崖边缘；禁止在首屏用大型按钮或卡片盖住角色；禁止固定全景镜头把角色留在远处；推进与旋转必须连续对应危险进度，不能使用随机抖动、景深虚化或突然缩放影响判距。
- 禁止用同一缩放掩盖角色体型差异，禁止让体重与速度数值违背体型和动作观感，禁止让商店覆盖层误触蓄力。
- 禁止把 HUD 做得比角色更亮，禁止结算层遮蔽坠落原因。
- 禁止让随机巨物变成清晰、饱和或带发光描边的主角；禁止它遮挡跑道、崖口、角色和距离标尺，也禁止添加彩蛋提示文字、奖励或突发惊吓音效。禁止每次渲染重新抽取导致模型跳变，禁止连续两局重复同一彩蛋。
- 跑道下方禁止圆锥、冰锥、圆柱、尖刺和自然岩根造型；厚度变化必须由边缘清晰的长短方块表达。禁止一整面等深方墙，也禁止所有方块长度相同。

## 11. 垂直切片验收

- 入口：无大型开始按钮，角色在英雄机位中完整可见；姓名、体重和速度使用无框三行文字，不覆盖角色、标题或 HUD；幽灵手指与短标签提示按住，且不覆盖角色面部、身体或脚部。“远征队”入口同时显示当前角色和永久金币。
- 游戏：至少 3 层冰壁、裂缝、冰柱和碎冰可辨；起步全景同时展示起点和崖口，镜头随危险进度从侧后方旋转推进到角色前侧近景，距断崖风险条不遮挡赛道。
- 收藏：两种体型差异显著的角色在同一机位中仍可一眼区分；购买、选择、余额不足和已拥有状态完整。
- 天气：晴朗、飘雪、薄雾、风雪四种状态均不遮蔽角色、崖口或操作提示。
- 高反馈：贴边停车同时具备后仰定格、距离数字、评价文字、暖光和冰晶粒子。
- 结束：先完整展示坠落原因，再出现包含本局分数和再来一次的结算层。
- 窄屏：320×568 下文字不截断、按钮不小于 44×44 px、HUD 与赛道不碰撞。

## 12. 屏幕与状态合同

- 开始/等待：玩家需要理解当前角色的体重、速度和“按住蓄力、松手出发”；焦点是角色、上方无框三行信息和底部蓄力槽，次动作是“远征队”。
- 蓄力：镜头保持英雄近景并轻微压近，三行角色信息持续可见，底部显示力度百分比；按住时间不受倒计时限制，1600 ms 后保持满值。
- 滑行：松手后隐藏三行角色信息与操作控件，顶部保留关卡、分数、金币，以及合并显示距断崖、天气、冰面阻力和进度条的小面板；全程不再接受游戏输入。自动刹车开始后面板边框与进度条转为珊瑚红。
- 停车结算：显示单关成绩、距离、金币、是否过关和下一位角色缩略图；不限时停留，必须由玩家点击“下一位角色”或“本关重试”，绝不自动开始。低分点击“本关重试”后仍停在角色起点近景与标准幽灵手指提示，直到玩家再次点击场景。
- 坠落：隐藏操作提示，保留 HUD，1250 ms 慢动作碎裂散落完成后出现结算层。
- 远征队：显示永久金币、收藏进度、全部 52 名角色、类别、体重、速度、价格和选择状态；关闭后回到打开前的封面或结算页。
- 游戏结束：显示本局分数、通过关数、本局金币与永久金币，主动作“再来一次”，次动作“远征队”和“返回封面”。
- 声音按钮固定右上角 48×48 px；无加载、联网、权限与社交状态。
