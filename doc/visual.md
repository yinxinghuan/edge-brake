# 《急刹车》视觉圣经

## 1. 视觉命题

- 游戏与受众：面向移动端碎片时间玩家的单手反应游戏，首局无需阅读说明。
- 情绪承诺：看着断崖持续逼近，在唯一一次点击前屏息判断，并在自动制动后等待结果揭晓。
- 一句话方向：一支可收藏的低多边形极地远征队在断裂冰舌上轮流挑战急停，每个体型、动作、天气和镜头变化都服务于“它还能滑多远”。
- 标志性瞬间：镜头贴近并绕向断崖侧面，角色后仰、手脚撑开、雪屑喷散，最终停在有冰柱与浪花的崖口 8 px 内。
- 必须具备：角色轮廓和体型差异清楚、冰崖有可信厚度与地貌层次、距离易判断、收藏入口不打断核心操作。
- 避免方向：写实摄影、廉价通用卡通 UI、霓虹玻璃科技感。

## 2. 构图与镜头

- 方向与比例：竖屏固定游戏场 390×700，3D 场景填满画布，DOM HUD 内部适配；同时验证 320×568 与 390×844。
- 开场镜头：封面加载后用 2.8 秒从 `(-6.2, 13.2, 18.4)`、zoom 29 的高位全景推进到 `(-2.5, 9.4, 13.6)`、zoom 38 的正式机位；观察点先扫过崖下海面，再回到角色与冰舌。镜头只播放一次，不循环。
- 游戏镜头：危险进度 0→1 时，OrthographicCamera 从 `(followX-2.5, 9.4, 13.6)` 推进至 `(followX-1.3, 7.8, 10.8)`，zoom 38→46，观察点额外前移 0.8u；X/Z 联动产生最多约 9°环绕。相机使用阻尼，不抖动，不把崖口移出视野。
- 主要焦点：画面中部 y=280–470 的当前角色、冰面与悬崖边缘。
- 层次：前景风雪、雪屑与近岸碎冰；中景角色、裂缝冰舌与立体断崖；背景为浮冰、冰柱、低对比冰山和天气层。
- HUD 安全区：顶部 28–112 px；底部 566–670 px 为教学/按压提示区，不遮挡赛道。
- 注意路径：角色前进方向 → 明亮冰崖切口与垂直冰壁 → 崖外海面 → 点击反馈与距离结果。

## 3. 色彩

- 天空背景：极夜蓝 `#10263B`；远景冰山 `#1F435A`；雪面 `#EAF9F7`；冰侧 `#A9DCD8`。
- 主文字：暖雪白 `#F7FBF6`；次文字 `#A9C2C5`；墨色轮廓 `#102129`。
- 角色：沿用素材库暖中性色、压灰服装色与统一青绿强调 `#3FB6AC`；企鹅继续使用炭黑、暖白与橙色。
- 操作强调：刹车珊瑚红 `#F06449`；贴边奖励黄 `#FFD166`；安全青绿 `#69D1B5`；失败红 `#E9504F`。
- 使用比例：天空与远景 55%，雪与冰面 25%，角色及轮廓 14%，强调色不超过 6%。
- 禁止：蓝底上用低对比蓝字；同时大面积使用珊瑚红和奖励黄；用颜色作为唯一评价信号。

## 4. 字体

- 展示与数字：`Arial Rounded MT Bold`, `Nunito`, `PingFang SC`, sans-serif；数字使用 tabular-nums。
- 正文与按钮：`Avenir Next`, `PingFang SC`, `Microsoft YaHei`, sans-serif。
- 标题 38/44 px、900；分数 30/34 px、800；按钮 18/22 px、800；辅助标签 13/18 px、700。
- 只允许分数与短标签全大写英文；中文不加宽字距，不使用描边文字。

## 5. 形状、材质与光线

- 主导形状：角色沿用素材库 box / low-segment cyl / icosa / cone / wedge 的体素语言；冰崖由错层盒体、楔形雪脊、六段冰柱和碎冰组成；UI 为 12–18 px 圆角的实心远征标签。
- 边缘：全部 3D 素材使用 flatShading 的硬切面，不加卡通描边；轮廓由三值光照与接触阴影建立。
- 材质：哑光低多边形材质，`metalness=0`、高粗糙度，不使用照片纹理或平滑高模表面。
- 光线：顶部主光、冷色填充光与暖色轮廓光形成 1.0/0.8/0.62 三值面；冰崖切面比顶面深 20–25%，角色和冰面必须有接触阴影。

## 6. 角色、环境与素材

- 角色阵容：企鹅、小孩、老奶奶、商务男、狐狸、青蛙、棕熊共 7 名；素材库模型保持原始比例与三值光照，通过统一落脚点和整体缩放进入游戏。矮胖、宽脚角色对应高抓地，瘦高或沉重角色对应低抓地。
- 姿态：待机以 0.06–0.10u 小跳和 4–6% 纵向拉伸表现呼吸；人体滑行时四肢交替摆动 18–30°，动物以身体起伏和前爪节奏表现；急停时后仰 12–18°、人体手臂后甩和腿前撑，动物压低 8–12%；坠落时手脚张开并绕 Z 轴旋转 38°。
- 环境：主冰舌至少包含 3 层厚度、2 条可见裂缝、3 组雪脊、5 根冰柱、4 块近岸碎冰和带浪脊的深海；顶面、侧壁、深层冰与海面形成四级明度。
- 天气：晴朗保持高对比；飘雪使用 120 个慢速点状雪片；薄雾缩短远景雾距但保留角色与崖口；风雪使用 180 个快速斜移雪点和更冷的环境光。约 650 ms 淡入，天气层跟随镜头。
- 游戏资产使用共享 low-poly 库的模型与同源图元，几何吸附 0.125u 网格，统一 flat-shaded 三值语言；不混用照片、平滑 3D 高模、2D 贴片或 emoji。

## 7. UI 与图标

- 图标：自绘 24 px SVG 线性图标，2.2 px 圆头描边；声音、金币、角色和关闭使用同一套轮廓规则并提供可访问名称。
- 主按钮：最小 168×52 px，珊瑚红实心、墨色 2 px 下边阴影；按下下移 3 px并缩放至 0.98。
- HUD：无大面板；顶部显示关卡、分数、金币，崖口接近时出现窄幅“距断崖”风险条；连击只在发生变化时短暂强调。
- 远征队：覆盖层使用暖白考察日志底板，顶部固定当前金币；角色以两列卡片排列，卡片包含库内缩略图、名称、抓地条、价格/已拥有/使用中。可滚动卡片使用 `onClick`，每个目标至少 92×126 px。
- 状态：焦点使用 3 px 暖雪白外框；禁用降至 42% 不透明度；警告同时用文字、姿态和红色；成功同时显示距离数字、评价文字和粒子。
- Emoji 规则：禁止用 emoji 作为功能图标或游戏角色素材。

## 8. 动效与特效

- 性格：短促、略带橡皮筋弹性；普通 UI 160–220 ms，关卡切换 420 ms。
- 开场：2.8 秒高位远景环绕推进，使用 ease-out-cubic；允许玩家随时点击开始，镜头在准备阶段接续到正式机位，不要求看完动画。
- 行进：镜头推进与危险进度连续绑定；角色待机、滑行与急停动作无缝过渡，四肢幅度随速度增加但不超过 30°。
- 输入：单击同帧锁定急停，角色按体型后仰 12–18°并压低；50 ms 内雪屑出现，操作提示立即切换为“制动中”，松手不改变状态。
- 收藏：购买成功后金币数字下跳、卡片印章由价格切换为“已拥有”，角色模型在 220 ms 内弹入当前选择；余额不足使用水平 4 px 轻摇并显示差额，不弹通用错误框。
- 停车：180 ms 定住，评价在 140 ms 内向上弹入；贴边额外有冰崖暖光和 8 枚旋转的低多边形冰晶。
- 失败：120 ms 悬停、530 ms 坠落并旋转 38°；画面只震动一次，位移不超过 5 px、时长 140 ms。
- 粒子：雪屑为 3–7 px 暖白椭圆；奖励冰晶为 4–8 px 黄白菱形，生命周期 220–480 ms。
- 减少动态：移除屏幕震动、背景风带和大量粒子；保留角色姿态、距离数字与评价文本。

## 9. 参考转译原则

- 《跳一跳》与《Block Hop》：采用待机小跳、四肢节奏、天气平滑轮换、金币收藏和即时重开；不复制道路、车辆、方块或原场景构图。
- 定点停车类街机：采用越接近危险边缘奖励越高的风险结构；不使用真实车辆仪表盘。
- 纸雕动画：采用色块分层和轻微材质划痕；不模拟复杂纸张噪声或厚重阴影。

## 10. 反模式

- 禁止 emoji、混合风格图标、写实冰川照片和平滑高模企鹅素材。
- 禁止玻璃拟态、霓虹光晕、大面积渐变、通用浮动白卡和无意义阴影。
- 禁止回到“单块白板”冰面；禁止让雪屑遮住冰崖边缘；禁止固定全景镜头把角色留在远处；推进与旋转必须连续对应危险进度，不能使用随机抖动、景深虚化或突然缩放影响判距。
- 禁止用同一缩放掩盖角色体型差异，禁止让抓地数值与体型观感相反，禁止让商店覆盖层在游戏中误触打开。
- 禁止把 HUD 做得比角色更亮，禁止结算层遮蔽坠落原因。

## 11. 垂直切片验收

- 入口：2.8 秒镜头依次交代当前角色、冰舌地貌、断崖高度和海面；“远征队”入口同时显示当前角色和永久金币。
- 游戏：至少 3 层冰壁、裂缝、冰柱和碎冰可辨；镜头随危险进度旋转推进，角色稳定在画面偏左，距断崖风险条不遮挡赛道。
- 收藏：两种体型差异显著的角色在同一机位中仍可一眼区分；购买、选择、余额不足和已拥有状态完整。
- 天气：晴朗、飘雪、薄雾、风雪四种状态均不遮蔽角色、崖口或操作提示。
- 高反馈：贴边停车同时具备后仰定格、距离数字、评价文字、暖光和冰晶粒子。
- 结束：先完整展示坠落原因，再出现包含本局分数和再来一次的结算层。
- 窄屏：320×568 下文字不截断、按钮不小于 44×44 px、HUD 与赛道不碰撞。
- 视觉 QA：第一轮 390×700 截图发现企鹅与冰崖被横向裁切，已通过统一 3D 坐标映射和镜头 zoom 修正；第一轮 320×568 截图发现 transform 缩放后右侧溢出，已改为视口中心绝对定位并复拍通过。证据保存在 `_qa/ui/`。
- 远征队改版 QA：首轮 390×700 发现雪帽外沿仍过直、素材人物侧面过薄、换关瞬间相机停留崖边；复验版已使用 5 段错位雪帽、角色三分之四朝向和结算期镜头回收修复。已覆盖企鹅/人物/动物、购买、晴/雪/雾/风雪、中文/英文及 320×568，证据保存在 `_qa/ui/expedition-v3/`。

## 12. 屏幕与状态合同

- 开始：玩家需要理解当前角色的抓地力、“会向右滑、点一下便锁定急停”；焦点是角色与断崖，主动作是“开始滑行”，次动作是“远征队”。
- 准备：500 ms 倒计时，以赛道短划线和企鹅前倾表示即将启动；准备阶段点击可预锁定急停。
- 滑行：顶部仅保留关卡、分数、金币；全画面单击一次锁定急停，底部从“点一下急停”切换到不可逆的“制动中”。
- 停车结算：冰崖上方显示距离与评价，850 ms 后自动进入下一关，无额外确认。
- 坠落：隐藏操作提示，保留 HUD，650 ms 动画后出现结算层。
- 远征队：显示永久金币、7 名角色、抓地条、价格和选择状态；关闭后回到打开前的封面或结算页。
- 游戏结束：显示本局分数、通过关数、本局金币与永久金币，主动作“再来一次”，次动作“远征队”和“返回封面”。
- 声音按钮固定右上角 48×48 px；无加载、联网、权限与社交状态。

## 13. 手感反馈矩阵

| 事件 | 即时反馈 | 结果与通道 | 强度 | 恢复 |
|---|---|---|---:|---|
| 开始 | 开场镜头接续推进、角色前倾、短促上滑音 | 500 ms 后进入正式跟随机位并开始滑行 | 1 | 进入游戏 |
| 单击急停 | 同帧后仰、身体压低、提示锁定 | 雪屑、刹车下降音、持续自动减速 | 3 | 等待停车或坠落，不可撤销 |
| 安全停车 | 角色回弹、显示距离 | 青绿标签与单音 | 2 | 850 ms 下一关 |
| 漂亮停车 | 较强回弹、两段提示音 | 黄白距离标签、较多雪屑 | 3 | 850 ms 下一关 |
| 贴边停车 | 70 ms 暖光、冰晶爆开 | 三音上行、连击增长 | 4 | 850 ms 下一关 |
| 坠落 | 120 ms 悬停、单次轻震 | 下降音、旋转坠落、失败文字 | 5 | 650 ms 结算 |
| 再来一次 | 按钮下压、短音 | 分数清零并回到准备态 | 1 | 立即开局 |
| 获得金币 | 金币数字上跳、金色粒子 | 150 ms 双音，永久余额更新 | 2 | 继续下一关 |
| 解锁角色 | 卡片印章翻转、角色弹入 | 三音上行、立即选中 | 4 | 关闭商店或继续浏览 |
| 余额不足 | 金币差额文字、卡片轻摇 | 90 Hz 短促闷音 | 2 | 保留商店位置 |

- 输入到视觉反馈目标小于 16 ms；输入到声音反馈目标小于 50 ms。
- 静音时全部关键反馈仍由姿态、文字和粒子表达。
- 减少动态时取消震动与背景循环，只保留不超过 220 ms 的必要状态过渡。
