# 《急刹车》需求文档

## 1. Overview

《急刹车》是一款选择不同抓地力的极地远征角色、观察断崖逼近并选择唯一急停时机、收集金币解锁队员的竖屏单击反应游戏，每局约 20–45 秒。

## 2. Visual Design

游戏采用 390×700 的竖屏 3D 构图。浮冰有雪脊、裂缝、分层冰壁、冰柱、碎冰和崖下海浪，有效跑道长度约为最初版本的 2.6 倍。首屏不放大型开始按钮，正交镜头以近距离三分之四角度突出当前角色，循环幽灵手指提示玩家直接点击角色附近；每一关都从角色近景开始，角色真正滑动后镜头才用 780 ms 拉远到能够完整看见起点、整条跑道和崖口的建立镜头。全景停留后镜头随危险距离连续推近并绕到人物前侧。所有角色身体和脚尖统一朝向跑道前进方向，不以斜站朝向镜头。天气在晴朗、飘雪、薄雾与风雪间轮换。封面和结算页提供“远征队”入口，用素材库角色缩略图、名称、抓地力和金币价格表达收藏进度。完整镜头、灯光、材质、色彩、字体、图标、动效与素材规则写入 `doc/visual.md`。

## 3. Game Mechanics

- 开局角色位于赛道左侧 x=40，初速度为 160 px/s；每一关先在近景机位准备 520 ms，准备期不接受刹车输入。角色进入自动滑动的同一帧开放刹车输入并启动镜头拉远：780 ms 到达全景，再固定停留 620 ms，角色在整个拉远与停留阶段持续移动。
- 每一关的悬崖边缘位于 x=760–820 之间随机变化；角色前缘越过悬崖边缘超过 12 px 即坠落并结束本局。起点到崖口有效滑行距离为 691–751 px，约为最初版本的 2.6 倍。
- 未触发急停时按跑道进度分三段变速：0–30% 为 65 px/s² 的快速起步，30–68% 为 160 px/s² 的光滑冰带冲刺，68–100% 为 95 px/s² 的临崖高速滑行；第 1 关最高速度为 390 px/s，此后每关增加 12 px/s，最高封顶 480 px/s。三段冰面以不同青蓝明度和短划纹区分，角色滑冰步频、身体压低幅度与速度同步。
- 玩家每关只有一次急停机会；点击后立即锁定制动状态并持续施加 `320 × 角色抓地系数` px/s² 的反向加速度，直到完全停止，不能取消或再次点击修正。
- 当速度降至 3 px/s 以下并持续 110 ms，判定停车完成，按角色前缘到悬崖边缘的距离结算：0–8 px 为“贴边”并得 5 分，9–22 px 为“漂亮”并得 3 分，23–52 px 为“安全”并得 1 分，大于 52 px 为“太早”且本关不得分。
- 每一关准备阶段都使用 zoom 64 的角色英雄机位，角色高度约占游戏场 14–18%；角色开始滑动后相机在 780 ms 内拉远到 zoom 16.5 的正式赛道全景，从角色所在的起点到尽头悬崖必须完整、连续可见，三段冰带、完整崖口及少量崖外海面同时进入构图，并固定停留 620 ms。拉远开始后的每一帧都接受急停输入；全景停留结束后，镜头按剩余危险距离连续推到 zoom 68、高度 5.0u 的人物近景，并沿 X/Z 方向绕到人物前侧；越过崖口后继续推至 zoom 78 的近正面坠落特写，不突然切镜头。
- 连续“贴边”形成连击：第 2 次起每次额外获得 `min(连击数 - 1, 5)` 分；非贴边停车会清空连击，但不会结束游戏。
- 角色阵容共 7 名：企鹅 1.00、小孩 1.12、老奶奶 0.92、商务男 0.84、狐狸 1.05、青蛙 1.22、棕熊 0.76。抓地系数越大，点击后的停车距离越短；角色体型、站姿和脚掌宽度需让强弱在场景中可预判。
- 企鹅默认解锁；单局首次到达第 3 关永久解锁小孩，第 5 关永久解锁狐狸。老奶奶、商务男、青蛙和棕熊分别花费 24、40、85、120 金币；已解锁角色可随时在封面或结算页选择。
- 每次成功停车获得金币：贴边 7、漂亮 4、安全 2、太早 1，另加 `min(贴边连击, 3)`；金币、已解锁角色、当前角色与最高到达关卡保存在本地。
- 天气按关卡轮换：第 1 关晴朗，第 2 关飘雪，第 3 关晴朗，第 4 关薄雾，第 5 关飘雪，第 6 关风雪，此后按 6 关周期重复。天气在 650 ms 内淡入，不改变摩擦数值，避免遮蔽角色差异。
- 成功停车后停留 850 ms 展示距离和评分，镜头在结算层后方先行返回起点近景；下一关角色在相同近景机位准备 520 ms 后才开始滑动与拉远，不能直接继承上一关的全景。
- 所有角色的面部、胸口和脚尖沿跑道 +X 方向站立。共享人物模型以本地 +Z 为正面并绕 Y 轴旋转 +90°；狐狸、青蛙和棕熊以本地 +X 为正面并保持 Y=0°，不得对所有模型套用同一朝向。首屏与每关准备镜头位于角色前侧 30–40°，能看见脸和胸口，同时角色本体仍严格朝向跑道。滑行前以 1.25 秒周期进行重心左右转移、呼吸与脚跟轻抬；点击后先用 180 ms 下蹲蓄力，再用 260 ms 向前伸展进入滑行。人体每 0.28–0.42 秒交替向后外侧蹬腿、单脚短暂滑行、双臂左右平衡；人物手臂与腿必须使用独立肩/髋枢轴。动物四足按本地 Z 轴完成沿 +X 的前后蹬地，并以本地 X 轴完成横向外展。急停分为 90 ms 重心前移、180 ms 双脚横向制动/双臂张开和阻尼回稳三段。坠落时人体双臂和双腿向四周张开、躯干同时绕 X/Z 轴翻滚；动物四足外展并蜷身翻滚；企鹅双鳍完全展开、双脚分开并翻身。刹车时循环显示 9 个低多边形雪屑粒子；贴边停车额外显示 8 枚绕冰崖旋转的低多边形冰晶。
- 最佳分数、最高连续贴边次数、金币、收藏和当前角色保存在本地；单局分数从 0 开始，永久金币不清零。

## 4. Controls

- Pointer / Touch：首屏在非功能按钮区域任意按下一次开始游戏；幽灵手指只是循环示范，不接收输入。游戏中任意位置按下一次立即锁定急停，后续按下与抬起不再改变制动结果。
- Keyboard：首屏按一次 Space 或 Enter 开始；滑行中按一次 Space 或 Enter 锁定急停。按键自动重复和松开事件不重复触发。
- 首屏无大型开始按钮；重新开始按钮使用单次指针按下。只有 520 ms 近景起步准备阶段不接受刹车输入；角色开始移动与镜头开始拉远的同一帧开放刹车，避免延迟玩家判断。
- 每关重新获得一次急停机会；停车结算、坠落和游戏结束阶段不接受游戏输入。
- “远征队”商店只在封面和游戏结束阶段打开；列表中的角色卡使用点击选择或购买，不触发游戏急停。

## 5. Win / Lose Conditions

- 游戏没有最终胜利，目标是连续通过更多关并刷新总分与贴边连击纪录。
- 角色前缘越过悬崖边缘超过 12 px 时失败；失败过程持续 650 ms，明确展示滑出、短暂悬停与坠落。
- 结算界面显示本局分数、通过关数、本局金币、永久金币、最佳贴边距离和历史最高分，并提供“再来一次”和“远征队”入口。
- 距离悬崖超过 52 px 的过早停车只记为本关 0 分，850 ms 后继续下一关，不直接结束游戏。

## 6. Sound Effects

- 开始滑行：180–260 Hz 的三角波上滑，时长 100 ms，音量 0.08。
- 触发急停：135 Hz 锯齿波在 150 ms 内下降至 82 Hz，音量 0.07；每关只触发一次，并与角色后仰及雪屑喷发对齐。
- 普通停车：320 Hz 正弦波短音，时长 90 ms，音量 0.10。
- “漂亮”停车：420 Hz 与 620 Hz 两个正弦波依次触发，总时长 180 ms，音量 0.12。
- “贴边”停车：520、780、1040 Hz 三音上行，间隔 55 ms，总时长 260 ms，音量 0.14。
- 坠落失败：160 Hz 锯齿波在 420 ms 内下降至 55 Hz，末尾叠加 80 ms 低频噪声，音量 0.14。
- 界面按钮：600 Hz 正弦波，时长 45 ms，音量 0.06；静音或音频初始化失败时不影响游戏。
- 获得金币：880 Hz 与 1180 Hz 正弦波相隔 45 ms，总时长 150 ms，音量 0.08；购买或解锁角色使用 420、620、840 Hz 三音上行，总时长 260 ms，音量 0.11。
