# 《急刹车》需求文档

## 1. Overview

《急刹车》是一款竖屏单手蓄力滑行游戏：玩家选择拥有不同体重与速度的 52 名角色，按住屏幕积蓄力度、松手出发，让角色依靠惯性滑到冰崖边缘并收集新队员，每局约 15–35 秒。

## 2. Visual Design

游戏保持 390×700 的低多边形极地 3D 世界、方块浮岛长跑道、可读的天气循环与远海巨物彩蛋。每关在封面/准备阶段给出带 SVG 天气符号的“本关天气”预报，在滑行 HUD 持续显示天气名称、明确的阻力变化和文字化驾驶提示；飘雪、薄雾和风雪必须让天空、远景能见度、近景粒子与画面边缘明显不同。标题封面不显示角色属性；进入准备与蓄力阶段后采用角色英雄近景，角色严格朝向跑道，体重与速度以无框、无底色的两行大号文字悬浮在角色上方留白，不再重复角色姓名。新角色解锁时，含姓名的解锁横幅先独占该区域约 1700 ms，横幅退场后属性才淡入；玩家提前开始蓄力时横幅立即让位。底部只保留“按住蓄力”的幽灵手指和力度槽。AlterU 内的封面与事故结算显示冠军头像、姓名和累计分数作为排行榜入口；点击后打开与冰崖考察日志同一视觉语言的完整单榜。按住时镜头缓慢压近，角色随力度持续下蹲、摆臂并在高力度区轻微颤动；松手瞬间角色蹬地冲出，相机跟随 280 ms 起跑特写，再用 920 ms 拉远至完整跑道全景并停留 900 ms，之后随角色接近崖口连续推近。成功、停得太远和坠崖继续使用三套不同镜头语言。完整色彩、材质、排版、镜头、动作与 UI 规则写入 `doc/visual.md`。

## 3. Game Mechanics

- 跑道起点为 x=40，悬崖边缘每关在 x=1820–1940 之间随机，角色前缘越过悬崖超过 12 px 即坠落。跑道顶部连续，底部由长短不同的方块冰体组成，最后 5% 保持薄悬挑。
- 封面、下一关与重试都停在不限时准备状态，不会自动出发。玩家按下屏幕后立即进入蓄力；蓄力从 0 增长至 1，满蓄力时间为 1600 ms，达到上限后保持 1，不循环、不回落。
- 力度使用非线性曲线 `power = 0.12 + 0.88 × (hold / 1600)^1.35`，范围为 0.12–1.00。按压不足 80 ms 仍按 80 ms 计算，避免误触产生完全静止的结果；超过 1600 ms 不继续增加。
- 每个角色新增体重与速度参数。体重为 18–240 kg，由素材体积、类别和体态推导；速度为 2.6–5.0，由腿长、动作类别和体型推导。二者在普通准备阶段显示于角色头顶；新角色解锁回合需等待 1700 ms 解锁横幅结束后再显示，并在远征队卡片中同时展示。
- 松手初速度为 `(335 + 365 × power) × speedFactor` px/s，其中 `speedFactor` 将速度 2.6–5.0 映射到 0.86–1.14。松手后不再接受游戏操作；较小的力度斜率用于扩大有效判断窗口。
- 滑行减速度为 `126 × surfaceFactor × (75 / weight)^0.22` px/s²，并限制在 92–166 px/s²。较重角色惯性保持更强，较快角色起步更猛，但两项均不直接等同于最终距离。
- 关卡表面系数按六关天气循环：晴朗 1.00、飘雪 1.05、薄雾 0.94、风雪 1.03。较大系数代表阻力更强：飘雪会较早停住，薄雾湿冰会滑得更远，风雪则为轻微增阻并有横向风雪视觉。变化保持在 ±6% 内，使 60 kg、速度 3.8 的默认角色在长跑道仍有可达的停车窗口。封面和等待/蓄力阶段显示“本关天气 + 效果”预报；滑行全程显示“距断崖 N px”、跑道进度条、天气名称、`+5% / −6% / +3%` 阻力变化及一句驾驶提示。自动刹车开始后边框和进度条转为珊瑚红强调。随机崖口距离与天气共同使同一角色每关所需按压时间不同。
- 松手后的前 360 ms 为助跑/蹬地爆发段，人物和双足动物使用快速步态，四足动物使用冲刺步态，幽灵使用前冲拉伸，机甲使用重踏；随后立即进入惯性滑行，肢体摆幅降至起步段的 8%，改为低幅度平衡动作。角色位置从松手首帧开始移动，不用动画延迟物理反馈。
- 松手后的滑行仍由系统持续减速，不增加第二次玩家输入；当 `当前速度 ÷ 当前减速度 ≤ 1.8 秒`，或按当前速度抵达断崖仅剩 1.3 秒时，自动进入可见刹车阶段。第二条是冲过头回合的保底，保证坠崖前也能完整看见角色在抢救。触发同帧恢复旧版刹车反馈：人形后仰并将双臂向后上方张开、双腿错位撑开，鸟类张翼、四足动物压低前爪，机甲使用重心后移；同时在脚后持续喷出 9 枚循环雪屑。刹车姿态保持到自然停止或坠崖，不能改变物理减速度与最终距离。
- 当速度低于 3 px/s 并持续 140 ms 时判定停止。角色前缘到崖口距离为 0–10 px 时得 100 分“贴边”，11–35 px 得 90 分“漂亮”，36–80 px 得 75 分，81–140 px 得 60 分，141–220 px 得 45 分，221–320 px 得 30 分；0–320 px 均通过。超过 320 px 得 20 分并重试本关。
- 完整滑行目标时长为 2.8–5.8 秒。若满蓄力和高速度角色越过崖口，则播放 140 ms 冻结和 1250 ms 慢动作碎裂；不使用完整模型侧翻或穿模动作。
- 成功停止进入 1550 ms 英雄镜头，随后显示不限时结算；停得太远进入 1450 ms 俯视测距镜头；坠崖进入垂直追随碎裂镜头。结算不会自动开始下一关或重试。
- 正式角色池保持共享素材库的 52 名角色：15 名普通人物、11 名文化职业、6 名怪物、5 名办公室角色、2 名精英角色、1 名机甲、1 名神话角色和 11 名动物。初始永久解锁“店主”，每次过关按清单顺序解锁并自动换上下一名角色；也可使用 12–260 金币提前购买。
- 100 分形成贴边连击；其他分数清空连击。金币奖励保持为 100/90/75/60/45/20 分分别获得 10/8/6/5/4/2 枚，另加 `min(贴边连击, 3)`。
- 最佳分数、最近停车距离、最高贴边连击、金币、收藏、当前角色和最高关卡保存在本地；玩法改版不清除已有收藏与金币。
- 排行榜按游戏永久 UUID 隔离，使用单一总榜；排行榜分数为一局从第 1 关开始累计、直到坠崖时的最终总分，平台只保留每名玩家的历史最高分。首次进入一局时快照玩家原最高分；坠崖进入事故结算时提交最终总分。若新纪录严格超过旧纪录，则刷新榜单，只通知本次刚被超过且分数最高的 1 名其他玩家；平分、未破纪录、自己或网络失败均不通知。
- AlterU 内封面和事故结算优先显示冠军入口：有榜单数据时显示第 1 名头像、姓名和分数，无数据时在同一位置显示“排行榜”。完整榜单最多展示平台返回的 50 行，包含排名、头像、姓名、分数和本人标识；点击其他玩家的头像姓名行打开其 AlterU 主页，本人行不可点击。加载、空榜和失败均保持可关闭，网络请求和通知不得阻塞开局、结算或重试。
- 天气按六关周期轮换：晴朗、飘雪、晴朗、薄雾、飘雪、风雪。天气层在预报出现时以 180 ms 入场，在发车后保持；飘雪使用不少于 240 枚慢雪点，薄雾用 6 组低多边形雾团与缩短能见度，风雪使用不少于 400 枚斜移雪点和画面边缘风痕；`prefers-reduced-motion` 时保留静态雾度/色温与文字提示，停用持续粒子运动。远海巨物池保持熊、牛、猪、狐狸、幽灵、僵尸、狼人、木乃伊、骷髅、战斗机甲与牛头怪；发车后 80–600 ms 从雾中显现，完整覆盖 1200–2100 ms 的最远全景与停留阶段，2300 ms 后才开始淡出，3400 ms 完全消失。

## 4. Controls

- Pointer / Touch：在封面或等待状态按住非功能按钮区域立即开始蓄力；保持按压持续增加力度；松开或发生 `pointercancel` 时按当前力度发车。滑行中再次按下不产生任何效果。
- Keyboard：在封面或等待状态按下 Space / Enter 开始蓄力，松开对应按键发车；忽略按键自动重复。滑行、结果镜头和结算阶段不接受蓄力输入。
- 远征队、声音、结算和返回按钮继续使用明确点击，必须阻止事件冒泡，不能意外开始蓄力或提前发车。
- 冠军入口使用点击打开排行榜；排行榜为纵向滚动区域，玩家行使用 `click` 区分轻触与滚动，关闭按钮与遮罩点击均可退出。排行榜打开时必须阻止底层蓄力、角色购买和结算按钮响应。
- 重新开始按钮只回到当前角色的等待近景；玩家必须重新按住并松开才能开始下一次尝试。

## 5. Win / Lose Conditions

- 游戏没有最终胜利，目标是连续通过更多关、收集 52 名角色，并学习不同体重、速度、天气和跑道长度对应的蓄力时长。
- 角色停在崖口前 0–320 px 内即通过；超过 320 px 为“动力不足”，说明本次按压太短，但保留本关 20 分与金币。
- 角色前缘越过崖口 12 px 为“冲过头”，进入慢动作碎裂失败；失败动画结束后才显示事故结算。
- 停车结算显示本关得分、距边缘距离、获得金币、下一名角色预告和下一步按钮；事故结算显示累计分数、通过关数、本局金币、最佳距离和永久金币。所有结算均不限时。
- 排行榜不改变过关或失败判定。平台提交失败时保留本地最高分和正常事故结算，玩家可立即重试；通知发送失败同样静默处理。

## 6. Sound Effects

- 开始蓄力：180 Hz 三角波在 140 ms 内上滑至 260 Hz，音量 0.06；每次按下只播放一次。
- 高力度提示：力度首次越过 0.82 时播放 520 Hz、55 ms 的短正弦音，音量 0.045，不循环。
- 松手发车：根据力度从 210–420 Hz 映射起音，在 120 ms 内上滑 1.45 倍，叠加 55 ms 低频冲击，音量 0.09–0.13。
- 自动刹车开始：首次进入最后 1.8 秒减速段时播放 260 Hz 方波在 150 ms 内下滑至 150 Hz，叠加 90 Hz、80 ms 的冰面摩擦低音；每回合只播放一次，音量 0.07–0.10。
- 天气预报：进入第 2 关及之后的等待状态时，飘雪播放 390 Hz、70 ms 的轻三角波，薄雾播放 240 Hz、110 ms 的低正弦波，风雪播放 180 Hz、130 ms 的低三角波；音量均不超过 0.045，静音时不播放。
- 自然停住：320 Hz 正弦波，90 ms，音量 0.10；“漂亮”使用 420/620 Hz 两音，总长 180 ms；“贴边”使用 520/780/1040 Hz 三音，总长 260 ms。
- 动力不足：210 Hz 方波接 118 Hz 锯齿下行，总长约 360 ms，音量 0.06–0.09。
- 坠崖：160 Hz 锯齿波在 420 ms 内下降至 55 Hz，末尾叠加 80 ms 低频噪声，音量 0.14。
- 界面按钮、金币、购买和解锁继续使用既有短音阶；静音或音频初始化失败不得阻塞输入与物理反馈。
- 打开或关闭排行榜沿用 180 Hz、80 ms 的轻界面短音，音量 0.05；排行榜加载、提交和通知不播放等待音，不得掩盖游戏结算音效。
