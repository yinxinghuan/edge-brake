# 《急刹车》需求文档

## 1. Overview

《急刹车》是一款观察悬崖逼近并选择唯一急停时机、争取让企鹅停在边缘最近位置的竖屏单击反应游戏，每局约 20–45 秒。

## 2. Visual Design

游戏采用 390×700 的竖屏 3D 构图。封面加载后的 2.6 秒内，正交镜头从高位远景轻微环绕并推进，同时交代企鹅、整块浮冰、断崖切面和海面；点击开始后在 500 ms 准备阶段自然收进斜俯视跟随机位。游戏中企鹅稳定在画面偏左区域，悬崖从右侧持续逼近，玩家始终看得到危险目标。HUD 只显示当前关数、累计分数和连续完美次数。首局通过自动滑行、单次点击示意和点击后的自动制动姿态完成无文字或极少文字教学。完整镜头、灯光、材质、色彩、字体、图标、动效与素材规则写入 `doc/visual.md`。

## 3. Game Mechanics

- 开局角色位于赛道左侧 x=64，初速度为 96 px/s，经过 500 ms 准备时间后开始自动向右移动。
- 每一关的悬崖边缘位于 x=330–350 之间随机变化；角色前缘越过悬崖边缘超过 12 px 即坠落并结束本局。
- 未触发急停时加速度为 70 px/s²；第 1 关最高速度为 238 px/s，此后每关增加 10 px/s，最高封顶 330 px/s。
- 玩家每关只有一次急停机会；点击后立即锁定制动状态并持续施加 260 px/s² 的反向加速度，直到完全停止，不能取消或再次点击修正。
- 当速度降至 3 px/s 以下并持续 110 ms，判定停车完成，按角色前缘到悬崖边缘的距离结算：0–8 px 为“贴边”并得 5 分，9–22 px 为“漂亮”并得 3 分，23–52 px 为“安全”并得 1 分，大于 52 px 为“太早”且本关不得分。
- 镜头在企鹅移动到画面 x≈140 后开始跟随，使企鹅保持在左侧 36% 附近；悬崖边缘、垂直冰壁和崖外海面必须持续进入前向视野。
- 连续“贴边”形成连击：第 2 次起每次额外获得 `min(连击数 - 1, 5)` 分；非贴边停车会清空连击，但不会结束游戏。
- 成功停车后停留 850 ms 展示距离和评分，然后立即开始下一关；企鹅以阻尼移动回出发位置，避免硬切跳变。
- 刹车时循环显示 9 个低多边形雪屑粒子；贴边停车额外显示 8 枚绕冰崖旋转的低多边形冰晶，持续到 850 ms 结算结束。
- 最佳分数和最高连续贴边次数保存在本地；单局分数从 0 开始。

## 4. Controls

- Pointer / Touch：在游戏区域任意位置按下一次立即锁定急停，后续按下与抬起不再改变制动结果。
- Keyboard：按一次 Space 或 Enter 锁定急停；按键自动重复和松开事件不重复触发。
- 开始与重新开始按钮使用单次指针按下；准备阶段的点击会预先锁定急停，并在滑行开始时立即制动。
- 每关重新获得一次急停机会；停车结算、坠落和游戏结束阶段不接受游戏输入。

## 5. Win / Lose Conditions

- 游戏没有最终胜利，目标是连续通过更多关并刷新总分与贴边连击纪录。
- 角色前缘越过悬崖边缘超过 12 px 时失败；失败过程持续 650 ms，明确展示滑出、短暂悬停与坠落。
- 结算界面显示本局分数、通过关数、最佳贴边距离、最高连击和历史最高分，并提供“再来一次”主按钮。
- 距离悬崖超过 52 px 的过早停车只记为本关 0 分，850 ms 后继续下一关，不直接结束游戏。

## 6. Sound Effects

- 开始滑行：180–260 Hz 的三角波上滑，时长 100 ms，音量 0.08。
- 触发急停：135 Hz 锯齿波在 150 ms 内下降至 82 Hz，音量 0.07；每关只触发一次，并与企鹅后仰及雪屑喷发对齐。
- 普通停车：320 Hz 正弦波短音，时长 90 ms，音量 0.10。
- “漂亮”停车：420 Hz 与 620 Hz 两个正弦波依次触发，总时长 180 ms，音量 0.12。
- “贴边”停车：520、780、1040 Hz 三音上行，间隔 55 ms，总时长 260 ms，音量 0.14。
- 坠落失败：160 Hz 锯齿波在 420 ms 内下降至 55 Hz，末尾叠加 80 ms 低频噪声，音量 0.14。
- 界面按钮：600 Hz 正弦波，时长 45 ms，音量 0.06；静音或音频初始化失败时不影响游戏。
