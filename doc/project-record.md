## 当前玩法

玩家从近景角色镜头出发，角色沿长冰舌自动加速滑向断崖。滑行开始后镜头先拉远展示完整跑道，再随危险距离推进；玩家拥有一次急停机会，需要根据角色可见体型与抓地力预判停车距离。停在距边缘 0–220 px 内通过，超过 220 px 判定为过早停车，越过崖口 12 px 则坠落失败。

每次停车按 100 / 90 / 75 / 60 / 45 / 20 六档评分并奖励金币。通过关卡会依次解锁并自动更换角色，也可在远征队商店提前购买。正式角色池共 52 名，覆盖人物、职业、怪物、办公室、精英、机甲、神话与动物八类，每名角色拥有 0.78–1.24 的独立抓地系数和对应动作分组。

## 已锁定的体验原则

- 每一关都从角色近景和点击等待开始；重试按钮只回到准备状态，不得自动发车。
- 角色实际滑动后，相机才在 1050 ms 内拉到能看完整起点、跑道和崖口的全景，并停留 1000 ms；此时急停输入始终有效。
- 人形和动物必须严格朝向跑道，镜头负责展示脸和身份特征，不可通过斜转角色迎合镜头。
- 有肢体角色需表现滑行动作、急停时的外撑与手臂后摆；坠落时使用慢动作碎裂，禁止完整模型翻转穿模。
- 成功、停得太远和坠崖必须使用三套可辨认的镜头语言，不能只更换结算文案。
- 结算页不限时停留，下一关或重试必须由玩家明确点击。

## 三种结果镜头

1. 成功停车：120 ms 定格后绕到角色正前侧并推至 zoom 78，配合分数、评价、冰晶爆发、光环和暖光；1550 ms 后进入不限时结算。
2. 停得太远：解释为“你停在了安全观景区，未进入断崖警戒线，挑战无效”。110 ms 定格后升高拉远至 zoom 7.4，完整展示人物、剩余跑道和崖口，并以珊瑚红测距虚线与 X 标记突出距离；1450 ms 后显示低分结算。
3. 坠崖失败：140 ms 冻结后角色碎成 14 块，镜头不横向绕人，而是沿垂直崖壁向下追随碎片，zoom 84→62；1250 ms 后显示事故结算。

## 技术与数据

- React 18 + TypeScript 5 + React Three Fiber / Three.js + Less + Vite 5。
- 构建基址固定为 `./`，所有发布资源保持相对路径，可部署到任意子路径。
- 物理由 `requestAnimationFrame` 驱动；速度、急停、评分与金币规则分别集中在 hook 与纯规则模块。
- 52 套 GLB 与 PNG 缩略图随项目构建，不依赖工作区外部资源。
- 本地记录键统一使用 `edge_brake_*` 前缀，包含最高分、最佳距离、最高贴边连击、金币、收藏、当前角色、最高关卡和静音状态。
- 当前尚未接入排行榜、云存档或后端接口；永久 UUID 已预留为未来平台数据的隔离锚点。

## 后续接手注意

- 改判定区间、速度或刹车距离时，同时复验不同抓地力角色，避免让 220 px 通过线重新变得过严。
- 改镜头时至少复拍 390×700 与 320×568；全跑道镜头必须同时看到人物和崖口，近景不能遮住分数或人物头部。
- 改角色模型时保留本地朝向约定：人形本地 +Z 旋转 +90°，动物本地 +X 保持 0°。
- 改结算流程时不得恢复自动开始；`result`、`gameover` 的推进仍需玩家点击。
- 上线前运行构建、UUID、相对路径检查；推送后必须检查线上 HTML 的新资源哈希和 bundle 特征字符串。
